# Memory

This file stores persistent information learned from interactions with the user.

## What to Remember

When you learn something about the user that might be useful in future conversations, use the `edit_file` tool to add it here.

Examples of things to remember:
- User's name, email, or other contact information
- Project details (tech stack, goals, current work)
- Preferences (coding style, communication style, tools)
- Context from previous conversations

## Examples

### User Profile
- Name: [User's name]
- Role: [Developer / Student / Manager / etc.]
- Location: [Timezone or location]

### Current Projects
- Project 1: [Brief description]
- Project 2: [Brief description]

### Preferences
- Programming language preference: [Python / JavaScript / etc.]
- Communication style: [Concise / Detailed / etc.]

## Notes

- Keep entries organized and easy to scan
- Remove outdated information
- This file is automatically loaded at the start of each conversation
